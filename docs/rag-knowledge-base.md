# RAG 知识库雏形

## 目标

让 AI 求职 Agent 不只依赖模型通用知识，而是能检索本地维护的 AI 产品求职知识，再基于检索结果进行简历分析、匹配判断和建议生成。

## 当前实现

当前版本是轻量本地 RAG：

1. 在 `src/data/knowledgeBase.ts` 中维护结构化知识片段；
2. 每条知识包含 `id`、标题、分类、标签和正文；
3. 前端根据当前 JD、简历和规则分析结果做关键词检索；
4. 将命中的知识片段注入 AI 增强 Prompt；
5. 要求模型在 `knowledgeCitations` 中返回实际参考的知识片段 ID；
6. 页面展示“RAG 检索参考”，让用户知道模型参考了什么。

## 知识分类

当前知识库覆盖：

- 岗位标准：AI 产品经理实习生的核心画像、落地推进要求；
- AI 产品方法：Agent、RAG、Prompt、Workflow、智能硬件等场景；
- 简历方法：AI 产品简历 Bullet 的结构和禁忌；
- 评测与 Badcase：指标、Badcase、幻觉风险、证据边界；
- 面试准备：常见追问方向；
- 作品集：GitHub、README、PRD、复盘材料。

## 为什么先不用向量库

当前阶段优先验证产品闭环，而不是追求技术复杂度：

- 本地关键词检索足够解释清楚“检索了什么”；
- 不需要额外数据库和 embedding 成本；
- 更适合开源初学者复现；
- 后续可以平滑升级为 embedding + 向量检索。

## 后续升级方向

1. 用 embedding 替代关键词检索；
2. 支持用户上传自己的 JD、优秀简历、面试经验作为知识库；
3. 对知识片段增加来源、可信度、适用岗位方向；
4. 将模型输出中的 `knowledgeCitations` 与页面卡片高亮联动；
5. 建立 RAG 评测集，检查检索是否命中正确知识。
