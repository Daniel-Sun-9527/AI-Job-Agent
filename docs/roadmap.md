# 迭代路线图

## 产品愿景

做一个开源、透明、可复用的 AI 产品求职训练工具，帮助学生和初级求职者更科学地理解岗位要求、优化简历、准备面试，并学习真实 AI 产品的构建方法。

## 设计原则

1. 先解决真实需求，再接更复杂的模型能力
2. 规则引擎保底，大模型增强，不让模型黑箱化
3. 默认保护隐私，所有公开样例必须脱敏
4. 所有建议必须可执行，不能只输出漂亮话
5. 文档和评测同步建设，方便他人学习和贡献

## V0.1：规则型 MVP

状态：已完成

- JD/简历输入
- 匹配度评分
- 六维能力雷达
- JD 要求匹配
- 简历建议
- 面试题
- Badcase 风险
- Markdown 导出

## V0.2：AI 增强准备层

状态：已完成

- 新增 AI 增强页
- 生成高质量 Prompt
- 明确隐私边界
- 中文化 README、PRD、评测文档
- 沉淀项目式学习记录

## V0.3：接入真实大模型

状态：已完成基础接入，待持续评测

已完成：

- 新增后端接口 `/api/enhance`
- 使用环境变量保存模型 API Key
- 支持 OpenAI-compatible API 或国内模型服务
- 模型输出结构化 JSON
- 前端展示 AI 增强建议

下一步：

- 对不同 JD 和简历样本运行 AI 增强结果
- 记录模型幻觉、建议空泛、证据引用不足等 Badcase
- 优化 Prompt 和输出字段
- 增加模型输出质量评分

关键风险：

- 不能把 API Key 写进前端
- 不能让模型编造经历
- 不能让输出变成不可解释的黑箱

## V0.4：评测集与 Badcase 看板

目标：

- 建立多 JD、多简历测试集
- 对比规则版和 AI 增强版
- 记录误判、遗漏、幻觉、建议空泛等问题
- 支持贡献者提交脱敏样例

## V0.5：开源协作与作品集包装

目标：

- 完成贡献指南
- 补充截图和演示视频
- 输出博客文章模板
- 支持一键导出作品集报告

## V1.0：公开发布

目标：

- 稳定部署
- 可公开试用
- 有完整样例数据和评测报告
- 面向 AI 产品求职者形成可持续贡献社区
