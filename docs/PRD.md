# PRD：AI 求职 Agent

## 1. 背景

AI 产品经理实习岗位正在从“传统产品助理”转向“懂 AI 能力边界、会做评测、能搭原型、能推动落地的 Builder 型产品实习生”。许多学生知道大模型、Agent、RAG 等概念，但很难把自己的经历转化成 JD 能识别的能力证据。

## 2. 目标用户

正在申请 AI 产品经理、AI Agent 产品经理、AIGC 产品经理、智能硬件 AI 产品经理、AI 平台产品经理等岗位的学生和初级求职者。

## 3. 用户问题

- 不知道自己的简历是否匹配某个 AI 产品岗位
- 不知道产品、AI、数据、Builder、项目推进等能力中哪一项最短板
- 简历描述过于泛泛，缺少“场景-行动-结果”的证据链
- 面试准备零散，无法围绕目标 JD 做针对性准备
- 作品集缺少产品闭环、评测框架和开源展示材料

## 4. 价值主张

帮助求职者把“目标 JD + 个人经历”转化成一份可执行的求职优化报告，包括匹配度、证据差距、简历改写、面试追问、项目补强路线和 Badcase 风险。

## 5. MVP 范围

### 输入

- 目标岗位
- 岗位 JD
- 当前简历或项目经历

### 输出

- 总体匹配度
- 六维能力雷达
- JD 要求逐项匹配
- 简历 Bullet 改写建议
- AI 产品经理面试题
- Badcase 风险
- AI 增强 Prompt 与模型增强结果
- AI 复核层：由大模型复核 JD 匹配、简历 Bullet、Badcase 风险
- AI 输出质检：结构完整性、证据扎根度、行动可执行性、幻觉风险控制
- Markdown 报告导出

## 6. 核心指标

- JD 要求覆盖率：多少关键要求能在简历中找到证据
- 建议可执行性：建议能否在 7 天内完成
- 证据具体度：建议是否包含项目、动作、工具、指标
- Badcase 召回率：是否能指出明显短板
- 面试相关性：面试题是否对应目标 JD
- AI 输出质检分：结构完整、证据扎根、可执行、低幻觉风险
- 首次报告生成时间：用户打开后 2 分钟内得到可用报告

## 7. 产品流程

1. 用户粘贴 JD 和简历
2. 系统抽取岗位要求和简历证据
3. 本地规则型 Agent 进行六维能力评分
4. 系统输出匹配证据、能力缺口、简历建议和面试题
5. 用户进入 AI 增强页，点击生成 AI 增强建议
6. 前端请求后端 `/api/enhance`，后端读取环境变量中的模型 Key 并调用 OpenAI-compatible API
7. 系统展示岗位画像、匹配优势、缺口优先级、简历 Bullet、面试追问、7 天补强计划和 Badcase 风险
8. 用户回到匹配诊断或简历优化页，查看 AI 复核后的 JD 匹配、简历 Bullet 和 Badcase 风险

## 8. 涉及的 AI 产品概念

- Agent Workflow：将复杂任务拆成多个可控步骤
- 信息抽取：从 JD 和简历中提取能力信号
- Prompt Engineering：把用户输入和规则分析结果组织成高质量模型指令
- LLM API Proxy：通过后端代理调用模型，避免浏览器暴露 API Key
- Structured Output：要求模型输出 JSON，便于前端稳定渲染和后续评测
- LLM-as-Feature Evaluation：对模型输出做结构、证据、可执行性和幻觉风险检查
- Evaluation：用指标判断建议是否真正有用
- Badcase Analysis：识别误判、遗漏、建议空泛等失败类型
- Human-in-the-loop：用户确认和修改模型输出，不自动伪造经历
- 隐私保护：真实简历进入模型前必须脱敏，开源样例必须使用脱敏或合成数据

## 9. 当前不做什么

- 不自动投递岗位
- 不存储用户真实简历
- 不在前端保存模型 API Key
- 不声称匹配分数等于真实录取概率
- 不替用户编造项目经历
- 不把模型输出直接视为事实，需要用户人工确认

## 10. 版本规划

- V0.1：规则型 Agent MVP
- V0.2：AI 增强 Prompt 与中文开源文档
- V0.3：后端模型代理和结构化 JSON 输出
- V0.4：评测集与 Badcase 看板
- V0.5：简历版本对比和作品集导出
- V1.0：公开发布并邀请更多求职者贡献脱敏样本
