# 评测样本集：JD 与项目经历

## 样本来源

本评测集由两部分组成：

1. JD 样本：来自用户提供的 AI 产品实习相关截图，已做脱敏和结构化整理。
2. 项目经历样本：参考公开简历与经验文章中的项目写法，只提炼结构和能力信号，不复制真实个人简历。

## 参考资料提炼

公开资料中反复出现的高质量项目经历写法：

- 用“数据 + 方法论 + 业务影响”表达产品价值；
- 每个项目要有明确目标、个人贡献、完整闭环和结果；
- 产品经理项目要覆盖需求发现、原型、优先级、上线、数据复盘；
- AI 产品项目要补充 Prompt、RAG、Agent、评测、Badcase、模型能力边界；
- 面向 AI 硬件、B 端、AIGC、电商、企业效率等方向时，需要突出对应场景的业务理解。

参考来源：

- ResumeAI《产品经理简历模板 2025》：强调产品简历要用“数据 + 方法论 + 业务影响”表达。
- Neat Stack《AI Product Manager Resume Example》：强调AI产品简历需要体现模型评估、A/B测试、RAG、Prompt、跨团队协作和可量化影响。
- PingCode《产品经理简历如何写项目经验》：强调项目背景、个人职责、量化成果、问题解决和团队协作。
- 萝卜简历《如何撰写产品经理项目经验》：强调从执行者转向产品负责人，用背景、任务、行动、结果表达项目价值。
- Adam Saulters AI PM Portfolio：体现AI作品集可以围绕Learn-Build-Polish、RAG、Prompt治理、Human-in-the-loop和ROI展开。

## 使用方式

这些样本用于测试 Agent 是否能做到：

1. 识别不同方向 JD 的能力重点；
2. 判断项目经历是否真的匹配；
3. 发现简历缺口，而不是只看关键词；
4. 生成可执行的项目补强建议；
5. 记录误判和漏判，形成 Badcase 看板。
