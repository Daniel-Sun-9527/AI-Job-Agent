# 贡献指南

欢迎为 AI 求职 Agent 贡献脱敏 JD、项目经历样本、Badcase 和改进建议。

## 可以贡献什么

1. 脱敏 JD 样本  
   删除公司内部信息、联系人、手机号、邮箱和个人隐私后提交。

2. 脱敏项目经历样本  
   不要提交真实姓名、学校、公司、导师、具体商业数据。

3. Badcase  
   记录系统判断不准、建议过泛、漏判、隐私风险或幻觉风险。

4. 产品改进建议  
   包括评测指标、页面交互、AI增强策略、开源文档。

## Badcase 提交格式

```md
### 问题类型
误判 / 建议过泛 / 漏判 / 隐私风险 / 幻觉风险

### 样本
使用了哪个 JD 或简历样本？

### 现象
你看到的具体问题是什么？

### 预期
你认为正确结果应该是什么？

### 可能根因
如果能判断，请写出可能原因。
```

## 从产品中导出 Badcase

1. 打开“Badcase”页签；
2. 记录或检查你的反馈；
3. 点击“生成 Issue”；
4. 将下载得到的 `badcase-issue.md` 内容粘贴到 GitHub Issue；
5. 如果需要批量分析，点击“导出 JSON”保存原始记录。

## 隐私要求

请不要提交以下内容：

- 真实姓名
- 手机号、邮箱、微信
- 学号、身份证号
- 未公开公司内部项目
- 未脱敏商业数据
- 真实面试官姓名或联系方式

## 贡献原则

- 优先帮助初学者理解 AI 产品求职，而不是堆砌复杂功能；
- 每个功能都应服务真实求职任务；
- 每个模型输出都要考虑评测、Badcase 和隐私边界；
- 中文文档优先，方便国内求职者使用。
