# AI Product Manager Internship Brief

Date: 2026-05-14

Candidate context:
- Chinese junior undergraduate student seeking AI product manager internship roles.
- Target companies: leading Chinese internet, AI, smart device, semiconductor, and consumer electronics companies.
- Target role cluster: AI product intern / AI Agent product intern / AIGC product intern / AI platform or model evaluation product intern.

Role signal summary:
- The strongest hiring signal is not "knows AI terms", but "can turn an AI use case into a tested product loop".
- Common JD keywords: user research, scenario analysis, requirement decomposition, PRD, prototype, cross-functional delivery, data analysis, model evaluation, user feedback, badcase analysis, prompt/RAG/Agent workflow, AI coding tools, MVP/demo.
- Strong bonus items: AI product/project internship, independent AI tool or Agent demo, SQL/Python/Excel analysis, Figma/Axure prototype, Dify/Coze/FastGPT/ComfyUI/Cursor/Claude Code/Copilot experience, GitHub/Notion/blog/portfolio.

Recommended positioning:
- "AI Agent / AI tool product intern with builder mindset": understands product basics, can use AI tools to build demos, can design model/product evaluation metrics, and can translate user pain points into product requirements.
- Primary portfolio project: an AI job-search assistant or vertical workflow Agent that includes JD ingestion, resume matching, mock interview Q&A, evaluation metrics, badcase log, PRD, prototype, and demo.
- Secondary portfolio asset: a C-end smart device or AI hardware scenario PRD, such as phone/car/home/glasses education or productivity Agent.

6-8 week learning path:
1. Week 1: Build JD matrix and product teardown habit. Analyze 30 AI PM JDs, use 10 AI products deeply, write 3 one-page teardowns.
2. Week 2: Learn practical AI literacy. Understand LLM, prompt engineering, RAG, Agent, tool calling, multimodal, model evaluation, hallucination, latency, cost, privacy.
3. Week 3: Product fundamentals. Practice user research, JTBD, scenario decomposition, PRD, flowchart, prototype, acceptance criteria.
4. Week 4: Build one demo. Use Dify/Coze/FastGPT or Cursor to ship a usable Agent/MVP and document the decisions.
5. Week 5: Data and evaluation. Build an evaluation set, label badcases, define metrics, run comparison tests, summarize improvement ideas.
6. Week 6: Application package. Produce one-page resume, portfolio page, PRD sample, prototype screenshots, demo link/GitHub, 20 interview answers.
7. Week 7-8: Targeted delivery. Customize applications by company direction: Byte/Alibaba/Baidu for Agent/platform/eval, Xiaomi/Huawei/OPPO/vivo for device ecosystem, Meitu/TapTap/Xiaohongshu-like roles for AIGC/content.

Resume rule:
- Every project bullet should follow: scenario/user -> problem -> product action -> AI method/tool -> metric/result.
- Avoid empty claims like "familiar with LLM". Replace with concrete proof: "built a RAG-based FAQ Agent with 80 test questions, classified hallucination and retrieval-miss badcases, redesigned prompts and retrieval filters."

Interview preparation anchors:
- Must prepare: why AI PM, favorite AI products and comparisons, AI reshaping phone/car/map/customer-service scenarios, how to improve LLM answer quality, RAG vs fine-tuning, Agent workflow design, model evaluation metrics, data-driven iteration, one project deep dive.
- Best answer style: translate technical terms into product language, then discuss user value, boundary conditions, evaluation, and iteration loop.
